The Kodu CLI by default starts the [TUI](/docs/tui) when run without any arguments.

```bash
kodu
```

But it also accepts commands as documented on this page. This allows you to interact with Kodu programmatically.

```bash
kodu run "Explain how closures work in JavaScript"
```

---

### tui

Start the Kodu terminal user interface.

```bash
kodu [project]
```

#### Flags

| Flag         | Short | Description                                                             |
| ------------ | ----- | ----------------------------------------------------------------------- |
| `--continue` | `-c`  | Continue the last session                                               |
| `--session`  | `-s`  | Session ID to continue                                                  |
| `--fork`     |       | Fork the session when continuing (use with `--continue` or `--session`) |
| `--prompt`   |       | Prompt to use                                                           |
| `--model`    | `-m`  | Model to use in the form of provider/model                              |
| `--agent`    |       | Agent to use                                                            |
| `--port`     |       | Port to listen on                                                       |
| `--hostname` |       | Hostname to listen on                                                   |

## Commands

The Kodu CLI also has the following commands.

---

### agent

Manage agents for Kodu.

```bash
kodu agent [command]
```

---

### attach

Attach a terminal to an already running Kodu backend server started via `serve` or `web` commands.

```bash
kodu attach [url]
```

This allows using the TUI with a remote Kodu backend. For example:

```bash
# Start the backend server for web/mobile access
kodu web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
kodu attach http://10.20.30.40:4096
```

#### Flags

| Flag        | Short | Description                       |
| ----------- | ----- | --------------------------------- |
| `--dir`     |       | Working directory to start TUI in |
| `--session` | `-s`  | Session ID to continue            |

---

#### create

Create a new agent with custom configuration.

```bash
kodu agent create
```

This command will guide you through creating a new agent with a custom system prompt and tool configuration.

---

#### list

List all available agents.

```bash
kodu agent list
```

### auth

Command to manage credentials and login for providers.

```bash
kodu auth [command]
```

---

#### login

Kodu is powered by the provider list at [Models.dev](https://models.dev), so you can use `kodu auth login` to configure API keys for any provider you'd like to use. This is stored in `~/.local/share/kodu/auth.json`.

```bash
kodu auth login
```

When Kodu starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
kodu auth list
```

Or the short version.

```bash
kodu auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
kodu auth logout
```

### github

Manage the GitHub agent for repository automation.

```bash
kodu github [command]
```

---

#### install

Install the GitHub agent in your repository.

```bash
kodu github install
```

This sets up the necessary GitHub Actions workflow and guides you through the configuration process. [Learn more](/docs/github).

---

#### run

Run the GitHub agent. This is typically used in GitHub Actions.

```bash
kodu github run
```

##### Flags

| Flag      | Description                            |
| --------- | -------------------------------------- |
| `--event` | GitHub mock event to run the agent for |
| `--token` | GitHub personal access token           |

### mcp

Manage Model Context Protocol servers.

```bash
kodu mcp [command]
```

---

#### add

Add an MCP server to your configuration.

```bash
kodu mcp add
```

This command will guide you through adding either a local or remote MCP server.

---

#### list

List all configured MCP servers and their connection status.

```bash
kodu mcp list
```

Or use the short version.

```bash
kodu mcp ls
```

---

#### auth

Authenticate with an OAuth-enabled MCP server.

```bash
kodu mcp auth [name]
```

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

```bash
kodu mcp auth list
```

Or use the short version.

```bash
kodu mcp auth ls
```

---

#### logout

Remove OAuth credentials for an MCP server.

```bash
kodu mcp logout [name]
```

---

#### debug

Debug OAuth connection issues for an MCP server.

```bash
kodu mcp debug <name>
```

### models

List all available models from configured providers.

```bash
kodu models [provider]
```

This command displays all models available across your configured providers in the format `provider/model`.

This is useful for figuring out the exact model name to use in [your config](/docs/config/).

You can optionally pass a provider ID to filter models by that provider.

```bash
kodu models anthropic
```

#### Flags

| Flag        | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `--refresh` | Refresh the models cache from models.dev                     |
| `--verbose` | Use more verbose model output (includes metadata like costs) |

Use the `--refresh` flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in Kodu.

```bash
kodu models --refresh
```

### run

Run kodu in non-interactive mode by passing a prompt directly.

```bash
kodu run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

```bash "kodu run"
kodu run Explain the use of context in Go
```

You can also attach to a running `kodu serve` instance to avoid MCP server cold boot times on every run:

```bash
# Start a headless server in one terminal
kodu serve

# In another terminal, run commands that attach to it
kodu run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flags

| Flag         | Short | Description                                                             |
| ------------ | ----- | ----------------------------------------------------------------------- |
| `--command`  |       | The command to run, use message for args                                |
| `--continue` | `-c`  | Continue the last session                                               |
| `--session`  | `-s`  | Session ID to continue                                                  |
| `--fork`     |       | Fork the session when continuing (use with `--continue` or `--session`) |
| `--share`    |       | Share the session                                                       |
| `--model`    | `-m`  | Model to use in the form of provider/model                              |
| `--agent`    |       | Agent to use                                                            |
| `--file`     | `-f`  | File(s) to attach to message                                            |
| `--format`   |       | Format: default (formatted) or json (raw JSON events)                   |
| `--title`    |       | Title for the session (uses truncated prompt if no value provided)      |
| `--attach`   |       | Attach to a running kodu server (e.g., http://localhost:4096)       |
| `--port`     |       | Port for the local server (defaults to random port)                     |

### serve

Start a headless Kodu server for API access. Check out the [server docs](/docs/server) for the full HTTP interface.

```bash
kodu serve
```

This starts an HTTP server that provides API access to kodu functionality without the TUI interface. Set `KODU_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `kodu`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |


#### list

```bash
kodu session list
```

##### Flags

| Flag          | Short | Description                          |

### stats

Show token usage and cost statistics for your Kodu sessions.

```bash
kodu stats
```

#### Flags

| Flag        | Description                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--days`    | Show stats for the last N days (all time)                                   |
| `--tools`   | Number of tools to show (all)                                               |
| `--models`  | Show model usage breakdown (hidden by default). Pass a number to show top N |
| `--project` | Filter by project (all projects, empty string: current project)             |

---

### export

Export session data as JSON.

```bash
kodu export [sessionID]
```

If you don't provide a session ID, you'll be prompted to select from available sessions.

---

### import

Import session data from a JSON file or Kodu share URL.

```bash
kodu import <file>
```

You can import from a local file or an Kodu share URL.

```bash
kodu import session.json
kodu import https://opncd.ai/s/abc123
```

---

### web

Start a headless Kodu server with a web interface.

```bash
kodu web
```

This starts an HTTP server and opens a web browser to access Kodu through a web interface. Set `KODU_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `kodu`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

---

### acp

Start an ACP (Agent Client Protocol) server.

```bash
kodu acp
```

This command starts an ACP server that communicates via stdin/stdout using nd-JSON.

#### Flags

| Flag         | Description           |
| ------------ | --------------------- |
| `--cwd`      | Working directory     |
| `--port`     | Port to listen on     |
| `--hostname` | Hostname to listen on |

---

### uninstall

Uninstall Kodu and remove all related files.

```bash
kodu uninstall
```

#### Flags

| Flag            | Short | Description                                 |
| --------------- | ----- | ------------------------------------------- |
| `--keep-config` | `-c`  | Keep configuration files                    |
| `--keep-data`   | `-d`  | Keep session data and snapshots             |
| `--dry-run`     |       | Show what would be removed without removing |
| `--force`       | `-f`  | Skip confirmation prompts                   |

---

### upgrade

Updates kodu to the latest version or a specific version.

```bash
kodu upgrade [target]
```

To upgrade to the latest version.

```bash
kodu upgrade
```

To upgrade to a specific version.

```bash
kodu upgrade v0.1.48
```

#### Flags

| Flag       | Short | Description                                                       |
| ---------- | ----- | ----------------------------------------------------------------- |
| `--method` | `-m`  | The installation method that was used; curl, npm, pnpm, bun, brew |

## Environment variables

Kodu can be configured using environment variables.

| Variable                              | Type    | Description                                       |
| ------------------------------------- | ------- | ------------------------------------------------- |
| `KODU_AUTO_SHARE`                 | boolean | Automatically share sessions                      |
| `KODU_GIT_BASH_PATH`              | string  | Path to Git Bash executable on Windows            |
| `KODU_CONFIG`                     | string  | Path to config file                               |
| `KODU_TUI_CONFIG`                 | string  | Path to TUI config file                           |
| `KODU_CONFIG_DIR`                 | string  | Path to config directory                          |
| `KODU_CONFIG_CONTENT`             | string  | Inline json config content                        |
| `KODU_DISABLE_AUTOUPDATE`         | boolean | Disable automatic update checks                   |
| `KODU_DISABLE_PRUNE`              | boolean | Disable pruning of old data                       |
| `KODU_DISABLE_TERMINAL_TITLE`     | boolean | Disable automatic terminal title updates          |
| `KODU_PERMISSION`                 | string  | Inlined json permissions config                   |
| `KODU_DISABLE_DEFAULT_PLUGINS`    | boolean | Disable default plugins                           |
| `KODU_DISABLE_LSP_DOWNLOAD`       | boolean | Disable automatic LSP server downloads            |
| `KODU_ENABLE_EXPERIMENTAL_MODELS` | boolean | Enable experimental models                        |
| `KODU_DISABLE_AUTOCOMPACT`        | boolean | Disable automatic context compaction              |
| `KODU_DISABLE_CLAUDE_CODE`        | boolean | Disable reading from `.claude` (prompt + skills)  |
| `KODU_DISABLE_CLAUDE_CODE_PROMPT` | boolean | Disable reading `~/.claude/CLAUDE.md`             |
| `KODU_DISABLE_CLAUDE_CODE_SKILLS` | boolean | Disable loading `.claude/skills`                  |
| `KODU_DISABLE_MODELS_FETCH`       | boolean | Disable fetching models from remote sources       |
| `KODU_DISABLE_MOUSE`              | boolean | Disable mouse capture in the TUI                  |
| `KODU_FAKE_VCS`                   | string  | Fake VCS provider for testing purposes            |
| `KODU_DISABLE_FILETIME_CHECK`     | boolean | Disable file time checking for optimization       |
| `KODU_CLIENT`                     | string  | Client identifier (defaults to `cli`)             |
| `KODU_ENABLE_EXA`                 | boolean | Enable Exa web search tools                       |
| `KODU_SERVER_PASSWORD`            | string  | Enable basic auth for `serve`/`web`               |
| `KODU_SERVER_USERNAME`            | string  | Override basic auth username (default `kodu`) |
| `KODU_MODELS_URL`                 | string  | Custom URL for fetching models configuration      |
